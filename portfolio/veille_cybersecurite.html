LA CYBERSÉCURITÉ DANS LE CLOUD COMPUTING

1️⃣ Contexte et enjeux

Le cloud computing repose sur des modèles de services IaaS, PaaS et SaaS, principalement fournis par des acteurs majeurs comme Amazon Web Services, Microsoft Azure et Google Cloud.

La migration des infrastructures vers le cloud modifie profondément le périmètre de sécurité traditionnel. On passe d’un modèle de sécurité périmétrique (on-premise) à un modèle basé sur :

  • La responsabilité partagée (Shared Responsibility Model)
  • La gestion des identités (IAM)
  • La sécurisation des API
  • La supervision continue

📌 Exemple : AWS Shared Responsibility Model
https://aws.amazon.com/compliance/shared-responsibility-model/

2️⃣ Principales menaces identifiées (2024–2026)

Selon les rapports de sécurité et les recommandations de l’ANSSI :

🔎 1. Mauvaise configuration des services cloud

  • Buckets S3 publics
  • Mauvaise gestion des règles de sécurité réseau (Security Groups)
  • Permissions IAM trop larges

🔎 2. Compromission d’identifiants

  • Attaques par phishing
  • Credential stuffing
  • Absence de MFA

🔎 3. Attaques sur les API

  • Exploitation de failles OWASP API Top 10
  • Manque d’authentification forte

🔎 4. Ransomware en environnement cloud hybride

🔎 5. Shadow IT

📚 Références :

Rapport ANSSI : https://www.ssi.gouv.fr
OWASP API Security : https://owasp.org/www-project-api-security/

3️⃣ Cadres normatifs et réglementaires

La cybersécurité cloud s’appuie sur plusieurs référentiels :

  • RGPD (protection des données)
  • ISO/IEC 27001
  • ISO/IEC 27017 (sécurité cloud)
  • ISO/IEC 27018 (protection des données personnelles dans le cloud)
  • NIST Cloud Security Framework

📚 Ressources :

CNIL (RGPD) : https://www.cnil.fr
NIST Cloud Computing : https://csrc.nist.gov/projects/cloud-computing

4️⃣ Bonnes pratiques techniques

🔐 1. IAM (Identity & Access Management)

  • Principe du moindre privilège
  • RBAC (Role-Based Access Control)
  • Rotation des clés d’accès
  • MFA obligatoire

🔐 2. Chiffrement

  • TLS 1.2 / 1.3 pour les données en transit
  • AES-256 pour les données au repos
  • Gestion sécurisée des clés (KMS)

🔐 3. Modèle Zero Trust

  • Vérification continue
  • Segmentation réseau
  • Authentification forte

🔐 4. Supervision et détection

  • SIEM (Security Information and Event Management)
  • SOC (Security Operations Center)
  • Solutions XDR/EDR

5️⃣ Outils et solutions étudiés

☁️ Sécurité native Cloud

  • AWS Security Hub
  • Microsoft Defender for Cloud
  • Google Security Command Center

🔍 Outils complémentaires

  • CASB (Cloud Access Security Broker)
  • WAF (Web Application Firewall)
  • IDS/IPS cloud
  • Outils d’audit de configuration (CSPM)

7️⃣ Analyse personnelle

La tendance actuelle montre un basculement vers :

  • Le modèle Zero Trust
  • L’automatisation de la sécurité (DevSecOps)
  • La détection basée sur l’IA
  • La sécurisation des environnements multi-cloud

La cybersécurité cloud devient une compétence stratégique pour les administrateurs systèmes et les ingénieurs cybersécurité.
