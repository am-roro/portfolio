SUJET : LA CYBERSÉCURITÉ DANS LE CLOUD

CRITIQUE

1️⃣ Contexte et enjeux

Le cloud computing repose sur des modèles de services IaaS, PaaS et SaaS, principalement fournis par des acteurs majeurs comme Amazon Web Services, Microsoft Azure et Google Cloud.

💡 Pour faire simple :
  • IaaS (Infrastructure en tant que Service)
    Le fournisseur vous loue les serveurs (le matériel). C'est comme louer un terrain vide : vous devez construire la maison et tout gérer vous-même.
  • PaaS (Plateforme en tant que Service)
    Le fournisseur s'occupe des serveurs et du système. C'est comme louer une maison déjà construite : vous n'avez plus qu'à amener vos meubles (vos applications).
  • SaaS (Logiciel en tant que Service)
    Tout est géré par le fournisseur, vous utilisez juste le service web. C'est comme aller à l'hôtel : tout est prêt, vous n'avez qu'à profiter (ex: Gmail, Netflix).

La migration des infrastructures vers le cloud modifie profondément le périmètre de sécurité traditionnel. On passe d’un modèle de sécurité périmétrique (on-premise) à un modèle basé sur :

  • La responsabilité partagée (Shared Responsibility Model)
  • La gestion des identités (IAM)
  • La sécurisation des API
  • La supervision continue
Exemple : AWS Shared Responsibility Model
